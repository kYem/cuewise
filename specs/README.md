# Specs

Accepted feature specifications for Cuewise. These documents describe approved features before implementation.

## Naming Convention

Files are numbered sequentially: `001-feature-name.md`, `002-another-feature.md`, etc.

## Status

| Status | Meaning |
|--------|---------|
| **Draft** | Work in progress, not yet reviewed |
| **Accepted** | Approved for implementation |
| **Implemented** | Feature has been built |
| **Superseded** | Replaced by a newer spec |
